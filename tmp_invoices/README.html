Chargebee

Invoices Export

Data in the exported files are comma separated and use the double quote (") character as the delimiter.

Note: Not all columns present in the exported files are explained here.

Invoices.csv

Invoice Number The invoice number. Acts as a identifier for invoice and typically generated sequentially.
Invoice Date The document date displayed on the invoice PDF. By default, it has the same value as the effective date of the action that created the invoice (subscription creation, update, or invoice creation). This date can be backdated (set to a value in the past) while performing the actions. Backdating an invoice is done for reasons such as booking revenue for a previous date or when the subscription or non-recurring charge is effective as of a past date. However, if the invoice is created as pending, and if the site is configured to set invoice dates to the date of closing, then upon invoice closure, this date is changed to the invoice closing date.
Start Date Start date of the invoice.
Status Current status of this invoice.
Next Retry Timestamp indicating when will the next attempt to collect payment for this invoice occur.
Recurring Boolean indicating whether this invoice belongs to a subscription
Tax Total Total tax amount for this invoice
Vat Number VAT/ Tax registration number of the customer. Learn more
PO Number Purchase Order Number for this invoice
Amount Due The unpaid amount that is due on the invoice. This is calculated as: total - amount_paid - sum of applied_credits.applied_amount - sum of adjustment_credit_notes.cn_total - sum of linked_taxes_withheld.amount.
Adjustments Total adjustments made against this invoice.
Credits Applied Total credits applied against this invoice.
Payments Payments collected successfully for the invoice. This is the sum of linked_payments[].txn_amount for all linked_payments[] that have txn_status as success.
Write Off Amount Amount written off against this invoice.
Net Term Days Number of days within which the invoice has to be paid
Due Date Due date of the invoice
Dunning Status Current dunning status of the invoice.

Date Indicates when this transaction occurred.
Payment Method The payment method of this transaction
Amount Amount for this transaction.
Status The status of this transaction.
Authorization Reason Type of authorization transaction
Error Code Error code received from the payment gateway on failure.
Error Text Error message received from the payment gateway on failure.
Gateway Gateway through which this transaction was done. Applicable only for 'Card' Payment Method
Id At Gateway The id with which this transaction is referred in gateway.
Reference Number The reference number for this transaction. For example, the check number when payment_method = check.

  • When the subscription is not on a contract term: this value is the number of billing cycles remaining after the current cycle, at the end of which, the subscription cancels.

Subscription Plan Amount in Decimal The decimal representation of the total amount for the plan, in major units of the currency. Always returned when multi-decimal pricing is enabled.

SubscriptionAddons.csv

Addon Id A unique ID for your system to identify the addon.
Addon Quantity Quantity of the addon. Applicable for addons with pricing_model other than flat_fee.
Addon Unit Price

The price or per-unit-price of the addon. The value depends on the type of currency.

Note:

For recurring addons, this is the final price or per-unit price for each billing period of the subscription, regardless of the addon period. For example, consider the following details:

  • The unit_price provided is $10
  • The addon billing period is 1 month.
  • The plan billing period is 3 months.
  • The addon is only billed for $10 on each subscription renewal.

Addon Amount In Decimal The decimal representation of the total amount for the addon, in major units of the currency. Always returned when multi-decimal pricing is enabled.

Addon Unit Price In Decimal

The decimal representation of the price or per-unit price of the addon. The value is in major units of the currency. Always returned when multi-decimal pricing is enabled.

Note:

For recurring addons, this is the final price or per-unit price for each billing period of the subscription, regardless of the addon period. For example, consider the following details:

  • The unit_price_in_decimal provided is $10.75
  • The addon billing period is 1 month.
  • The plan billing period is 3 months.
  • The addon is only billed for $10.75 on each subscription renewal.
Addon Current Term Start Date Start of the current billing period of the Addon.

Addon Next Billing Date The date/time at which the next billing for the Addon happens.

SubscriptionContracts.csv

Contract Start Date The start date of the contract term
Contract End Date The end date of the contract term
Billing Cycles The number of billing cycles of the subscription that the contract term is for.
Action At Term End Action to be taken when the contract term completes.
Created At The date when the contract term was created.
Status Current status of contract
Cancellation Cutoff Period The number of days before contract_end, during which the customer is barred from canceling the contract term. The customer is allowed to cancel the contract term via the Self-Serve Portal only before this period. This allows you to have sufficient time for processing the contract term closure
Total Amount Raised Total amount raised as an invoice for this subscription in this contract
Total Contract Value The sum of the totals of all the invoices raised as part of the contract term. For active contract terms, this is a predicted value. The value depends on the type of currency. If the subscription was imported with the contract term, then this value includes the value passed for total_amount_raised.
Total Amount Raised Before Tax It represents the cumulative sum of all the invoices raised for a specific contract term up until the point of importing the subscription. This amount does not include any applicable taxes and is therefore exclusive of tax. It's important to note that this value is typically used to calculate the total value of the contract before taxes, which is referred to as total_contract_value_before_tax.
Total Contract Value Before Tax It refers to the total amount of revenue that is expected to be generated from a specific contract term, calculated as the sum of all invoices raised during the term, regardless of payment status. It is based on past performance and the specified currency in the contract. If the subscription was imported, the value for total_amount_raised_before_tax is included in the calculation of the total contract value before tax. It's important to note that this value excludes any applicable taxes.

SKU The field is used as Product name/code in your third party accounting application. Chargebee will use it as an alternate name in your accounting application.
Accounting Code This field is to capture the Account code setup in your Accounting system for integration purposes only.
Accounting Category1 Used exclusively with the following accounting integrations
  • Xero: If you’ve categorized your products in Xero, provide the category name and option. Use the format: <category-name>: <option>. For example:Location: Singapore.
  • QuickBooks: If you’ve categorized your product sales in QuickBooks according to Classes, provide the class name here. Use the following format: <parent class>:<sub-class-1>:<sub-class-2>...
  • NetSuite: If you’ve categorized your products in NetSuite under Classes, provide the class name here. Use the following format: <parent class>: <sub-class-1>: <sub-class2>.... For example: Services: Plan.
  • Intacct: If you’ve classified your products in Intacct under Locations, provide the name of the Location here.
Accounting Category2 Used exclusively with the following accounting integrations
  • Xero: If you’ve categorized your products in Xero, then provide the second category name and option here. Use the format: <category-name>: <option>.... For example, Region: South
  • QuickBooks: If you’ve categorized your product sales in QuickBooks according to Location, provide the Location name here. Use the following format: <parent-location>:<sub-location-1>:<sub-location-2>.... For example: Location: North America: Canada
  • NetSuite: If you’ve categorized your products in NetSuite under Locations, provide the location name here. Use the following format <parent-location> : <sub-location-1>: <sub-location-2>.... For example: NA:US:CA
  • Intacct: If you’ve classified your products in Intacct under Dimensions, provide the value of the Dimension here.
Accounting Category3 Used exclusively with the following accounting integrations
  • NetSuite: If you’ve categorized your products in NetSuite under Departments, pass the department name here. Use the following format: <parent-department> : <sub-department-1>: <sub-department-2>.... For example: Production: Assembly.
  • Intacct: If you’ve classified your products in Intacct under multiple Dimensions, provide the value of the second Dimension here.
Accounting Category4 Used exclusively with the following accounting integrations
  • NetSuite: Provide the “Revenue Recognition Rule Id” for the product from NetSuite.
  • Intacct: If you have configured “Revenue Recognition Templates” for products in Intacct, provide the template ID for the product.
price_in_decimal The price of the plan when the pricing_model is flat_fee. When the pricing model is per_unit, it is the price per unit quantity of the item. Not applicable for the other pricing models. The value is in decimal and in major units of the currency. Also, this is only applicable when multi-decimal pricing is enabled.
free_quantity_in_decimal The quantity of the plan that is available free-of-charge, represented in decimal. When a subscription is created for this plan or when the plan of a subscription is changed to this one, only the quantity above this number is charged for. Applicable for quantity-based plans and only when multi-decimal pricing is enabled.

Price in Decimal The price of the addon when the pricing_model is flat_fee. When the pricing model is per_unit, it is the price per unit quantity of the item. Not applicable for the other pricing models. The value is in decimal and in major units of the currency. Also, this is only applicable when multi-decimal pricing is enabled.

This price is for the period of the addon. When attached to a plan, the addon is billed for the billing period of the plan. Learn more is enabled.

Coupons.csv

Coupon Id

Used to uniquely identify the coupon in your website/application and to integrate with Chargebee.

Note:

When the coupon ID contains a special character; for example: #, the API returns an error. Make sure that you encode the coupon ID in the path parameter before making an API call.

Name

The display name used in web interface for identifying the coupon.

Note:

When the name of the coupon set contains a special character; for example: #, the API returns an error. Make sure that you encode the name of the coupon set in the path parameter before making an API call.

Invoice Name Display name used in invoice. If it is not configured then name is used in invoice.
Discount Type The type of deduction
Discount Percentage The percentage of the original amount that should be deducted from it.
Discount Quantity Number of offered quantity for discount.
Duration Type Specifies the time duration for which this coupon is attached to the subscription.
Duration Month (Deprecated) The duration of time in months for which the coupon is attached to the subscription. Applicable only when duration_type is limited_period.
Note: This parameter has been deprecated. Use period and period_unit instead.
Period The duration of time for which the coupon is attached to the subscription, in period_units. Applicable only when duration_type is limited_period.
Period Unit The unit of time for period. Applicable only when duration_type is limited_period.
Max Redemptions

Maximum number of times this coupon can be redeemed.

Note:

If not specified, the coupon can be redeemed an indefinite number of times.

Valid From The date from which the coupon can be applied to subscriptions.
Valid Till Date upto which the coupon can be applied to new subscriptions.
Redemptions The number of times this coupon has been redeemed.
Status Status of the coupon.
Archived At Timestamp indicating when this coupon was archived.
Created At Timestamp indicating when this coupon is created.
Apply On The amount on the invoice to which the coupon is applied.
Plan Constraint Plans the coupon can be applied to.
Addon Constraint Addons the coupon can be applied to.
