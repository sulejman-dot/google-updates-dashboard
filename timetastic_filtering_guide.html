Processing: Map userId to Email

Step 1: Get all users from Module 8A-1 (25 users with email mapping)

Step 2: For each absence, lookup userId → email

absent_emails = ["cosmin@seomonitor.com", "claudiu@seomonitor.com", "delia@seomonitor.com"]

Processing: Filter Logic

For each user in Module 8:

  • Cosmin: "cosmin@seomonitor.com" IN absent_emails → EXCLUDE ✗
  • Claudiu: "claudiu@seomonitor.com" IN absent_emails → EXCLUDE ✗
  • Delia: "delia@seomonitor.com" IN absent_emails → EXCLUDE ✗
  • Diana: "diana@seomonitor.com" NOT in absent_emails → INCLUDE ✓
  • Andrei: "andrei.p@seomonitor.com" NOT in absent_emails → INCLUDE ✓

Final Calculation

Total Active Users (from Google Sheets): 5
Absent Users (from Timetastic): 3
Available Users (after filtering): 2
If 2 users checked in today:
Response Rate = (2 / 2) × 100 = 100%
Without Timetastic integration:
Response Rate = (2 / 5) × 100 = 40%

✅ Improvement: +60% accuracy!

🛡️ Edge Cases Handled

1. No Absences Today

Timetastic Response: Empty absences array []

absent_emails: Empty array []

Result: All active users pass through filter ✓

2. All Team Members Absent

Timetastic Response: All users in absences

absent_emails: Contains all user emails

Result: Empty filtered array, response_rate = 0% ✓

3. Partial Day Absences

Timetastic Response: User has absence from 14:00-17:00

Filtering Logic: User is still marked as absent for the day

Result: User excluded from available count ✓

4. API Failure

Error Handling: Module 8A set to "Continue on error"

Fallback: If API fails, absent_emails = empty array

Result: All users counted (same as before integration) ✓

5. Email Mismatch (case sensitivity)

Problem: Google Sheets has "Alice@company.com", Timetastic has "alice@company.com"

Solution: Use lower() function in filter condition

Result: Case-insensitive matching ✓
